Deploynode serve.js

No build step, no bundler, no framework. All JavaScript is loaded as native ES modules directly in the browser. There is no npm install step — the only runtime dependency is Node.js for the server, which handles static file serving, a CORS proxy for proxied external APIs, and the NLCD raster tile filter.

Self-Hosting

Bay Hive runs on any machine with Node.js 18 or later. No cloud account, no database, no build pipeline.

  1. Clone the repository: git clone https://github.com/ChrisMenning/habitat
  2. Start the server: node serve.js
  3. Open http://localhost:3000 in your browser.

Two external data sources (Cornell eBird and USDA National Agricultural Statistics Service) require free API keys for their respective layers to load. A NOAA CDO key is optional — it enables live frost-risk alerts but is not needed for the Climate Ribbon, which loads temperature normals and 2021–present observed data from bundled and IEM sources. The app works fully without any keys — keyed layers simply will not appear. Create an api-keys.txt file in the project root with one KEY=value line per key. See the README for details.

Extending & Adapting

Bay Hive is built with a modular architecture that makes it straightforward to add new data sources, refine alert algorithms, or adapt the tool for a different region.

⌥
Adding Data Sources

New layers are added by creating a new JS module that fetches and normalizes data, registering a MapLibre layer in map.js, and adding config entries. No build step or toolchain setup required.

◷
Trend Harvesting

Organizations running Bay Hive as a hosted service can schedule recurring observation snapshots via POST /api/harvest. Snapshots accumulate in snapshots/ and power the year-over-year trend charts in the Trends pane once at least two years are collected.

⌖
Adapting for Another Region

The center coordinate (CENTER in config.js), analysis radius, and the GBCC corridor data source are the primary region-specific values. Replacing the corridor data source and repointing the center coordinate is the minimum fork for a new geography.

⚡
Alert Algorithm Tuning

All seventeen alert algorithms are in alerts.js and documented in plain language. Distance thresholds, score cutoffs, and classification rules are named constants at the top of each function. They can be adjusted without touching any other file.

Open Source Governance

Bay Hive has no formal organizational home yet. These are the three realistic paths to sustainable stewardship for an open-source conservation tool.

Option A

Fiscal Sponsorship

A 501(c) fiscal host administers funds while the project retains operational independence. Enables grant applications and tax-deductible donations without forming a new nonprofit. Candidates include Open Source Collective and Aspiration Tech.

Option B

Local Nonprofit Stewardship

The codebase and development transfer to an existing organization — most naturally the Green Bay Conservation Corps (GBCC). The cleanest model: the people doing the corridor work maintain the tool that serves it.

Option C

Purpose-Built Organization

A governing body with seats for GBCC, a conservation nonprofit, a local government partner, and a technical maintainer. Higher setup cost but appropriate if the tool expands to serve multiple regions.
